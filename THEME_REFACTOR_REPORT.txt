IOTA ML centralized theme refactor
==================================

Scope
-----
- Inspected every frontend CSS file, including Cubes.css.
- Added frontend/src/styles/theme.css as the final stylesheet import.
- Centralized dark, light, shared, responsive, category, table, modal,
  button, icon, status, radius, border, and compatibility tokens.

Design changes
--------------
- Replaced the harsh black/white contrast with calm slate surfaces and a
  muted teal accent in both modes.
- Removed shadows, glows, and backdrop blur.
- Made icon-only controls transparent and borderless; hover changes color only.
- Reduced nested framing: one border for major regions, surface contrast for
  nested cards and controls, and subtle separators only where necessary.
- Simplified modal columns, result cards, MUI DataGrid rows, fields, and inputs.
- Kept category colors distinct but less saturated for long sessions.
- Added responsive theme spacing/radius variables.

Centralization results
----------------------
- Raw color literals outside theme.css: 423 -> 0
- Custom-property declarations outside theme.css: 275 -> 0
- Raw border-radius values outside theme.css: 221 -> 0
- Active shadows/glows outside theme.css: 128 -> 0
- Active backdrop filters outside theme.css: 24 -> 0
- Total !important declarations: 2274 -> 1305
- Exact duplicate declarations: 0

Maintenance safeguards
----------------------
- Added npm run check:theme.
- Added the theme contract check to npm run check.
- Updated frontend/src/styles/README.md with editing rules.
- Updated the root README with theme customization instructions.

Validation
----------
- TypeScript typecheck: passed
- Stylelint: passed
- Theme contract check: passed
- CSS debt check: passed
- Vite production build: passed

Theme check guarantees that feature CSS contains:
- no raw HEX/RGB/HSL colors
- no custom-property declarations
- no raw radius values
- no active shadows
- no active backdrop blur
