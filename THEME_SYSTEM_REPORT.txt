IOTA ML semantic theme refactor
===============================

Scope
-----
- Scanned every frontend CSS, TS, and TSX source file.
- Replaced every generated, legacy, or compatibility color reference with a semantic theme role.
- Removed --tone-*, --radius-ref-*, --iota-*, --minimal-*, --workflow-shell-*, --results-*, --n8n-modal-*, and --ai-* compatibility references.
- Removed raw color literals from all frontend source files outside theme.css.
- Moved React-controlled defaults for project colors, plot colors, and the login cube illustration into theme.css.
- Removed obsolete shadow declarations from feature CSS.
- Kept only runtime layout/data variables such as --project-color, --swatch, and modal column widths outside the theme contract.

Theme source of truth
---------------------
frontend/src/styles/theme.css

The file contains:
- dark and light palettes
- primary, secondary, status, text, surface, divider, control, table, icon, and overlay roles
- button roles
- node and category colors
- radii, border widths, responsive theme spacing, and component dimensions
- project/plot/login illustration colors consumed by React

Icon buttons
------------
Icon-only controls are globally enforced as:
- background: transparent
- border: 0
- no shadow or glow
- theme icon color with a color-only hover state
